Q: Consistency model
| Name                 | Summary                                      |
|----------------------+----------------------------------------------|
| Weak consistency     | After a write, reads may or may not see it   |
| Eventual consistency | After a write, reads will eventually see it  |
| Strong consistency   | After a write, reads will immediately see it |

- Weak consistency: Used in scenarios like VoIP; live online video; Realtime multiplayer games, etc
- Eventually consistency: Used in search engine indexing; DNS; Amazon S3; SimpleDB, etc
- Strong consistency: Used in filesystem; RDMBS, etc
